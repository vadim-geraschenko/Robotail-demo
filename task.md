# Задача для Codex: демо интерактивной карточки игрового автомата Robotail

## 1. Что нужно сделать

Собери короткое standalone-демо на React: современную интерактивную карточку товара с ROI-калькулятором для автомата:

**Автомат морской бой «Бродяги морей»**

Цель демо — показать, как обычную карточку дорогого B2B-оборудования можно превратить в понятный продающий экран: не просто “фото + цена + описание”, а “что это, почему зарабатывает, как быстро может окупаться”.

Это демо для генерального директора, менеджеров и SMM/контент-команды, а не для технических специалистов. Поэтому видимая часть должна говорить коротко, бизнесово и понятно. Не надо объяснять стек, InSales, архитектуру, Liquid, CMS и т.п. в интерфейсе.

---

## 2. Жёсткий скоуп

Это демо должно быть реально сделать за вечер.

Не делай огромную production-страницу. Нужна компактная, эффектная демонстрация.

Сделать:

1. Hero-блок с товаром, ценой, опциями и CTA.
2. Галерею из 3–5 изображений.
3. ROI-калькулятор окупаемости.
4. 3–4 карточки “почему автомат зарабатывает”.
5. Короткий блок характеристик.
6. Финальный CTA.

Не делать:

- корзину;
- регистрацию;
- полноценный каталог;
- CRM;
- интеграцию с InSales;
- backend;
- сложную админку;
- 10+ секций;
- длинные SEO-тексты;
- технические объяснения для разработчиков;
- автозвук;
- тяжёлую 3D-графику.

## 7. Структура интерфейса

### 7.1 Hero

Слева:

- крупное фото автомата;
- миниатюры галереи снизу или сбоку;
- бейджи: `Новинка`, `Предзаказ`.

Справа:

- H1: `Автомат морской бой «Бродяги морей»`
- короткий лид:

`Двухсторонний аркадный автомат с механикой двойной оплаты, призовым модулем и QR-бонусами. Для ТРЦ, игровых клубов и развлекательных центров, где важны трафик, вовлечение и повторные игры.`

- цена: `630 000 ₽`
- чекбоксы опций;
- итоговая цена с учётом опций;
- кнопки:
  - `Рассчитать окупаемость`
  - `Оформить предзаказ`

Кнопка “Рассчитать окупаемость” должна плавно скроллить к калькулятору.

---

### 7.2 ROI-калькулятор

Это главный блок демо.

Заголовок:

`Посчитайте окупаемость под свою площадку`

Подзаголовок:

`Цена автомата — это только вход. Важнее понять, сколько он может приносить при вашем трафике.`

Поля-слайдеры:

1. Цена игры  
   - default: `200 ₽`  
   - range: `50–500 ₽`

2. Оплат в день  
   - default: `80`  
   - range: `10–400`

3. Дней работы в месяц  
   - default: `30`  
   - range: `15–31`

4. Аренда места в месяц  
   - default: `30 000 ₽`  
   - range: `0–200 000 ₽`

5. Обслуживание и призовой фонд  
   - default: `15 000 ₽`  
   - range: `0–150 000 ₽`

Формулы:

```ts
monthlyRevenue = pricePerGame * paymentsPerDay * daysPerMonth
monthlyProfit = monthlyRevenue - rent - serviceCost
paybackMonths = totalPrice / monthlyProfit
annualRevenue = monthlyRevenue * 12
```

Если `monthlyProfit <= 0`, показывать:

`При этих параметрах модель не окупается. Попробуйте увеличить трафик или цену игры.`

Показывать крупными карточками:

- `Выручка в месяц`
- `Прибыль после расходов`
- `Окупаемость`
- `Выручка за год`

Добавить маленький дисклеймер:

`Расчёт ориентировочный. Реальная окупаемость зависит от локации, трафика, цены игры и условий обслуживания.`

## 8. Тон текста

Писать коротко, уверенно, без канцелярита.

Не надо:

- “инновационный инновационный”;
- длинные абзацы;
- технический жаргон;
- “мы внедрим CRM/AI/автоматизацию”; 
- обещания гарантированной прибыли.

Надо:

- “может приносить”;
- “примерная окупаемость”;
- “подходит для ТРЦ и игровых клубов”;
- “помогает монетизировать проходной трафик”;
- “сценарий покупки становится понятнее”.

## 9. UX

- Страница должна выглядеть отлично на desktop и нормально на mobile.
- На mobile hero должен идти сверху вниз: фото → название/цена → опции → CTA → калькулятор.
- Все интерактивные элементы должны быть очевидными.
- Значения ROI должны пересчитываться мгновенно.
- Слайдеры должны быть удобными.
- Избранная опция должна сразу менять итоговую цену.
- Никаких alert/window prompt.
- Никакого автозвука.
- Учитывай `prefers-reduced-motion`.

---

## 10. Главное впечатление от демо

После просмотра за 30–60 секунд должно быть понятно:

1. Товар стал выглядеть дороже и современнее.
2. Клиент быстрее понимает, зачем ему автомат.
3. Карточка продаёт через экономику, а не только через описание.
4. Есть интерактив, который помогает менеджеру получить заявку.
5. Автор демо понимает и текст, и визуал, и пользовательский сценарий.

---

## 11. Критерии готовности

Демо готово, если:

- проект запускается через `npm install && npm run dev`;
- есть современный hero с реальным фото товара;
- галерея работает;
- опции меняют итоговую цену;
- ROI-калькулятор работает;
- результаты красиво анимируются;
- есть 4 карточки преимуществ;
- есть характеристики;
- есть финальный CTA;
- мобильная версия не разваливается;
- визуально это выглядит заметно сильнее стандартной карточки магазина;
- код не раздут и понятен.
